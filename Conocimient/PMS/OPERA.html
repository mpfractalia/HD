OPERA

 

Opera (Oracle) se diferencia de Sihot en que el programa es accesible mediante un acceso directo a IE, apuntando al servidor de Opera del hotel o al DC de Alemania (la mayoría de hoteles europeos fuera de España no tienen servidor propio de Opera, se conectan a

https://mhs01-opera5-prod.int.microsdc.com:4401/OperaLogin/Welcome.do) y una vez hecho el login, al abrir PMS se abre un applet Java.

La página de inicio de sesión de Opera es la siguiente:

 

Al hacer click en PMS deberemos elegir la propiedad a la que acceder con un pop up en hoteles que trabajan en DC, y finalmente tendremos acceso al programa:

Para instalar Opera tenemos un paquete en Software homologado, básicamente instala varios complementos:

 

Micros Fidelio Opera Print Control Opera JinitCheck Control

Opera Register Terminal Oracle Jinitiator 1.3.1.25

Para abrir Opera también se necesita tener instalados Java y Adobe Reader (Las versiones difieren dependiendo si el hotel tiene Opera 5.04 o 5.05)

 

También podemos instalar Opera accediendo des de IE a la página de login del programa des de un usuario administrador y se solicitará la instalación de los complementos automáticamente.

 

Las acciones que haremos des de FE dentro de Opera serán las siguientes (procedimentado):

 

-Altas de usuario auditor

-Configuración de impresoras y tareas de impresión (print tasks)

 

La gestión de usuarios de Opera también deberá ser realizada por el keyuser (jefe/a de recepción/administración), en el caso que no estén disponibles se deberá pasar al grupo funcional de Opera.

 

Como en el caso de Sihot, otros tipos de incidencias funcionales de Opera deberán ser documentadas (preferentemente con captura de pantalla) y escaladas a TEF-SD- OPERA.
